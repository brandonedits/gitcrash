Configure GitHub

  • git config --global username "Brandon Hempel"
  • git config --global useremail "brandonhempel@gmail.com"

Initialize Local Repository

  • git init
  • ls: list files/folders

Files names

  • U = Untracked
  • A = Added (to staging area)

Status

  • get status = git status
  • add all files/folders = git add .
  • add specific files = git.add index.html
  • add to local reposity = git commit -m 'initial commit'
  • see history of commits = git log ('q' = quit)

Pull Changes from Remote Repository (GitHub)

  • Pull changes = git pull

IGNORE file

Shortcuts

  • git commit -am 'added Shorcut add/commit at the same time'
  • git add index.html && git commit -m 'add 1 file/commit at the same time' (Does not appear to work on Windows using the VSCode's terminal)

Getting Code From GitHub

  • Downloading: Download zip of the repo
  • git clone: Get a copy of the full repo on your machine
  • git pull: Get the latest changes from a repo
  • git fetch: Get latest changes without merging
  • forking: Copy a repo into your own account

Branching - login

  • Branching: Allows you to work on new features or bug fixes without affecting the main code base.
  • Create branches for different environments: Production, staging, development
  • create new branch: git checkout -b feature/login (b = branch)
  • See what branch you are on: git branch
  • push to login branch: git push -u origin login
